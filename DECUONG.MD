## 1. Mô tả đề tài
Website bán hàng được xây dựng nhằm hỗ trợ người dùng tìm kiếm, lựa chọn và mua sản phẩm trực tuyến một cách thuận tiện.  
Hệ thống cho phép quản trị viên quản lý sản phẩm, danh mục, đơn hàng và người dùng.

---

## 2. Mục tiêu
- Xây dựng website bán hàng hoàn chỉnh
- Áp dụng kiến thức về lập trình web (Frontend + Backend)
- Làm việc nhóm và sử dụng GitHub để quản lý mã nguồn

---

## 3. Đối tượng sử dụng
- Quản trị viên (Admin)
- Khách hàng (User)

---

## 4. Chức năng hệ thống

### 4.1. Admin
- Quản lý người dùng
- Quản lý danh mục sản phẩm
- Quản lý sản phẩm
- Quản lý đơn hàng
- Quản lý tin tức
- Thống kê và báo cáo

---

### 4.2. User
- Đăng ký / đăng nhập
- Xem sản phẩm
- Tìm kiếm sản phẩm
- Xem chi tiết sản phẩm
- Thêm vào giỏ hàng
- Đặt hàng
- Thanh toán online (VNPay, MoMo,...)
- Quản lý đơn hàng
- Cập nhật thông tin cá nhân

---

## 5. Công nghệ sử dụng
- Frontend: HTML, CSS, JavaScript
- Backend: PHP (Laravel)
- Cơ sở dữ liệu: MySQL
- Quản lý mã nguồn: GitHub

---

## 6. Phân công công việc

### Thành viên 1: Backend (User + Authentication)
- Xây dựng chức năng đăng ký, đăng nhập, đăng xuất
- Xử lý xác thực và phân quyền
- Quản lý thông tin cá nhân người dùng
- Xây dựng chức năng đặt hàng
- Quản lý đơn hàng phía user

---

### Thành viên 2: Backend (Admin + Database)
- Thiết kế cơ sở dữ liệu
- Xây dựng chức năng quản trị:
  - Quản lý người dùng
  - Quản lý danh mục
  - Quản lý sản phẩm
  - Quản lý đơn hàng
- Xây dựng API cho frontend
- Thống kê và báo cáo

---

### Thành viên 3: Frontend (Giao diện + UX/UI)
- Thiết kế giao diện website
- Xây dựng:
  - Trang chủ
  - Trang sản phẩm
  - Trang chi tiết
- Xây dựng giỏ hàng và thanh toán
- Kết nối API từ backend
- Thiết kế responsive

---

## 7. Kế hoạch thực hiện

- Tuần 1: Phân tích yêu cầu, thiết kế database
- Tuần 2: Xây dựng chức năng cơ bản (login, sản phẩm)
- Tuần 3: Hoàn thiện chức năng nâng cao (giỏ hàng, thanh toán)
- Tuần 4: Kiểm thử và sửa lỗi
- Tuần 5: Hoàn thiện báo cáo và demo
